{% extends "base.html" %} {% block content %} When using the harness as a JSON API, all requests must specify the Content-Type headers as 'application/json'. It is also possible to script the normal form endpoints, but the response will be HTML, so this is not recommended.

Single Endpoint Usage:
Curl:
curl -X POST http://localhost:5000/single -d '{"dwc:decimalLongitude": 5, "dwc:decimalLatitude": 6}' -H "Content-Type: application/json"
{ "dwc:decimalLatitude": 6.0, "dwc:decimalLongitude": 5.0, "flags": [] }

Batch Endpoint Usage:
Curl:
curl -X POST http://localhost:5000/batch -d '{"items":[{"dwc:decimalLongitude": 5, "dwc:decimalLatitude": 6},{"dwc:decimalLatitude": 5.5,"dwc:decimalLongitude": 7.1}]}' -H "Content-Type: application/json"
{ "items": [ { "dwc:decimalLatitude": 6.0, "dwc:decimalLongitude": 5.0, "flags": [] }, { "dwc:decimalLatitude": 5.5, "dwc:decimalLongitude": 7.1, "flags": [] } ] }
